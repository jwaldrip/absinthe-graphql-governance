# Contributing to Absinthe GraphQL Governance

Thank you for your interest in contributing to the Absinthe GraphQL Governance! We welcome contributions from everyone and are grateful for your help.

## How to Contribute

### Proposals

1. **Drafting a Proposal:**
   - Fork the `absinthe-graphql/governance` repository.
   - Create a new Markdown file in the `/proposals` directory with the filename `[github-username]-[proposal-name]-MM-DD-YYYY.md`.
   - Include a detailed description of the problem, proposed solution, impact assessment, and implementation plan.
   - Submit a pull request (PR) to the `exploration` branch.

2. **Review and Discussion:**
   - Participate in the discussion thread of your PR to address concerns and refine the proposal.
   - The review period typically lasts 2 weeks to ensure ample time for feedback.

3. **Voting:**
   - After the review period, maintainers will initiate a voting comment on the PR thread.
   - Use GitHub reactions to cast your vote (üëç for positive, üëé for negative).
   - A 2:1 positive to negative feedback ratio is required for the proposal to pass.

4. **Implementation:**
   - Once approved, the proposal is assigned to contributors for implementation.
   - Track implementation progress in the PR thread.

5. **Adoption:**
   - After successful implementation and testing, the proposal is moved to the `/adopted` folder.

### Reporting Issues

1. **Bug Reports:**
   - Submit bugs via GitHub Issues in the relevant repository.
   - Provide a clear and detailed description of the issue, including steps to reproduce, expected behavior, and actual behavior.

2. **Feature Requests:**
   - Submit feature requests via GitHub Issues or as a proposal following the proposal process above.

### Code Contributions

1. **Fork the Repository:**
   - Fork the repository you want to contribute to and clone it locally.

2. **Create a Branch:**
   - Create a new branch for your work (e.g., `feature/my-new-feature`).

3. **Write Tests:**
   - Ensure your code includes tests to cover new functionality or bug fixes.

4. **Submit a Pull Request:**
   - Push your branch to your fork and submit a pull request to the `main` branch.
   - Ensure your PR description includes the purpose of the change, relevant issue numbers, and any additional information needed for review.

5. **Code Review:**
   - Participate in the code review process by addressing feedback and making necessary changes.

## Code of Conduct

By participating in this project, you agree to adhere to the [Code of Conduct](CODE_OF_CONDUCT.md) that promotes respect, inclusivity, and constructive collaboration.

## Thank You

Thank you for contributing to the Absinthe GraphQL Governance! Your efforts help ensure the success and sustainability of the project.
